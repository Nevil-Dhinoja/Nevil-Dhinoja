---

AI engineer based in Gujarat, India. I build applied AI systems that go beyond proof-of-concept into structured, deployable pipelines — agents that reason under failure, retrieval systems that degrade gracefully, data pipelines that are honest about what they loaded and why.

My work sits at the intersection of LLM engineering, retrieval systems, and data engineering. A recurring theme across projects is the gap between a system that works in the happy path and one that holds up when something breaks — bad input, a missing database, a corrupt tool response, a rate-limited API. I run deliberate break experiments on everything I build and document the failures before moving on.

This is where I keep the work. Browse the repos.
